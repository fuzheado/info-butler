# Verify-before-assert directive

My training data has a knowledge cutoff. The AI/software landscape —
model names, API pricing, Docker images, SaaS products, framework
versions — moves fast. Everything I "know" about what exists or doesn't
exist may be stale.

Before I make any factual claim (positive or negative) about:
- Whether a model, tool, Docker image, or product exists
- What a product's capabilities, pricing, or API look like
- Whether a technical approach is feasible with current tools

...I MUST first search the web or fetch official documentation to verify.
Unverified assertions about what does or doesn't exist are not acceptable
in this project.
